## 核心需求（按优先级，凌驾于其他工作约定之上）

1. **信息安全**：用户的隐私是这个系统最高保护对象。所有功能设计先过安全关，不暴露给第三方、不污染日志。涉及凭证 / 个人信息 / 数据库访问的改动，事前主动评估泄露面，不等用户提醒。

- 每次开始一个新任务前，先执行 `git add -A && git commit -m "checkpoint: [任务描述]"`
- 先说明计划，询问用户是否可以，用户同意后再申请修改，不允许擅自做任何决定
- 每次用户提出需求时，先和用户讨论方案，确认符合需求，不要直接开始修改，也不要加入基础功能外的限制，全透明和用户商量
- 如果同一个问题用户强调多次，不要怀疑用户，你的一切判断以客观代码情况为准
